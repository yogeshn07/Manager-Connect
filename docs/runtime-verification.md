### Build Failure: Native Asset Path Issue

```
'C:\Users\YOGESH' is not recognized as an internal or external command,
operable program or batch file.

Building native assets for package:objective_c failed.
```

**Root cause:** The `objective_c` 9.4.1 package (transitive dependency: `supabase_flutter` → `passkeys` → `passkeys_darwin` → `objective_c`) runs a build hook that fails when the user's Windows profile path contains a space (`C:\Users\YOGESH N\`). The build hook command splits the path at the space, treating `C:\Users\YOGESH` as the executable and `N\...` as a separate argument.

**This is not a code issue.** It is a known Flutter/Windows environment issue with the native assets build system.

**Dependency chain:**
```
supabase_flutter 2.15.0
  └── supabase 2.13.0
        └── gotrue 2.22.0
              └── passkeys 2.20.0
                    └── passkeys_darwin 0.4.0
                          └── objective_c 9.4.1  ← build hook fails on spaced paths
```

**Impact:** The `objective_c` package is only needed for iOS/macOS Passkeys support. Android and web builds don't use it — but Flutter's native asset system compiles ALL package hooks regardless of target platform.

### Workarounds (not applied — environment-level)

| Workaround | Description |
|-----------|-------------|
| **Recommended** | Build on a CI system (GitHub Actions) where the path has no spaces |
| Alternative 1 | Move the Flutter SDK and project to a path without spaces (e.g., `C:\dev\flutter`) |
| Alternative 2 | Use a Windows junction/symlink: `mklink /J C:\dev\project "C:\Users\YOGESH N\OneDrive\Desktop\office_project"` |
| Alternative 3 | Set `PUB_CACHE` to a path without spaces: `set PUB_CACHE=C:\PubCache` then re-run `flutter pub get` |
